## Remediation Strategy

### 1. Low Risk / Patch Updates (PR-B)
Focus on simple `cargo update -p <package>` for transitive dependencies with available fixes and low breaking risk.
- **Scope**: `bytes`, `rsa`
- **Verification**: `cargo test --workspace`, `nix run .#prverify`

### 2. Medium Risk / Minor Updates (PR-C)
Update dependencies that might have minor breaking changes or require ensuring compatibility with parent crates (e.g., `url` -> `idna`).
- **Scope**: `idna` (via `url`?)
- **Verification**: Ensure no `url` related breakages.

### 3. High Risk / Breaking Changes (PR-D & PR-E)
Handle direct dependencies that require code changes or major version bumps.
- **Scope**: `git2` (0.19 -> 0.20), `atty` (Replace)
- **Verification**: Full manual verification of CLI functionality.
